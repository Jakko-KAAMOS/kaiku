Setup

Open eval/reaper-template.rpp. Kaiku is pre-loaded on the MIDI track. Render settings: 48 kHz, 24-bit, stereo. Do not change them. Use headphones or nearfield monitors. Enable REAPER's spectrum analyser on the master bus before you render.

What you're listening for

  • The slow beat (wheel carrier, ~1.008×). It changes period with register. Wait for it.
  • The reverb tail in the rests. That's half the test.
  • The b2 — G♮ — in context. It should pull toward F#.
  • Any click, pop, or artifact on attack or release.
  • In the spectrogram: harmonic series, FM sideband pairs, noise floor cleanliness.

Inspecting renders

After each render: load the WAV into REAPER's media explorer and view with the spectrum analyser. Look for clean harmonic series, consistent FM sidebands, and a flat noise floor. Aliasing shows as spikes above 15 kHz. The wheel carrier beating is visible as paired sidebands at ±1.008× the fundamental.

The HTML report

One entry per exercise. Fields: expected, observed, rating (Pass / Marginal / Fail), free notes. Marginal = you heard something but can't characterize it. Fail = artifact, wrong pitch, silence, pop. If you're not sure, it's Marginal. Write what you heard.

Submit

  • Completed HTML report
  • WAV renders for any Marginal or Fail exercise
  • Spectrogram screenshots for anomalies

Do not submit

  • REAPER project files
  • Spectrogram screenshots for passing exercises
  • Aesthetic opinions — this is functional testing
  • Revised or corrected MIDI

What this means for Phase 2

Phase 1 passes → Ensemble calibration and LFO parameter testing.
b2 exercises fail → Operator rebalancing before anything else.
Envelope artifacts → ADSR parameter review and voice-reset audit before Phase III.
